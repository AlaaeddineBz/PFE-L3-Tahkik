# Chapter Template Generator

## Context

This skill generates a single chapter `.tex` file for the Tahkik PFE mémoire. It produces well-structured LaTeX using the project's design system — tcolorbox styles, section formatting, figure/table environments, and cross-referencing conventions. The output integrates seamlessly with the scaffold generated by the `latex-scaffold` skill.

## Instructions

When invoked, generate a chapter file with the following structure:

### Chapter File Structure

```latex
% ════════════════════════════════════════════════════════════════════════════
%  CHAPTER <N> — <CHAPTER TITLE (UPPERCASE)>
% ════════════════════════════════════════════════════════════════════════════
\chapter{<Chapter Title>}
\label{ch:<label_prefix>}

<Chapter introduction paragraph — 2-3 sentences explaining what this chapter covers>

\section{<Section Title>}
\label{sec:<label_prefix>_<section_slug>}

<Section content>

\subsection{<Subsection Title>}
\label{subsec:<label_prefix>_<subsection_slug>}

<Subsection content>
```

### Using tcolorbox Styles

Use the project's box styles for specific content types:

**Key definitions or concepts:**
```latex
\begin{emeraldbox}[title=<Concept Name>]
<Definition or key concept explanation>
\end{emeraldbox}
```

**Highlights, planned features, or notable points:**
```latex
\begin{goldbox}[title=<Highlight Title>]
<Important point or planned feature>
\end{goldbox}
```

**Warnings, limitations, or critical notes:**
```latex
\begin{warnbox}[title=<Warning Title>]
<Warning or limitation description>
\end{warnbox}
```

**General information or context:**
```latex
\begin{infobox}[title=<Info Title>]
<Contextual information>
\end{infobox}
```

**Process steps or phases (use for workflows/pipelines):**
```latex
\phaseblock{1}{<Phase Name>}{
  <Description of what happens in this phase>
}
```

### Figure Environments

When a figure is needed, use this pattern:

```latex
\begin{figure}[H]
  \centering
  % \includegraphics[width=0.8\textwidth]{figures/<label_prefix>_<figure_name>.png}
  \fbox{\parbox{0.7\textwidth}{\centering\vspace{2cm}
    \textit{Figure: <Description>}\vspace{2cm}}}
  \caption{<Caption text>}
  \label{fig:<label_prefix>_<figure_slug>}
\end{figure}
```

For inline TikZ diagrams (preferred for UML/architecture), generate the TikZ code directly instead of using `\includegraphics`.

### Table Environments

When a table is needed, use this pattern:

```latex
\begin{table}[H]
  \centering
  \caption{<Caption text>}
  \label{tab:<label_prefix>_<table_slug>}
  \renewcommand{\arraystretch}{1.3}
  \begin{tabular}{lll}
    \toprule
    \textbf{Column 1} & \textbf{Column 2} & \textbf{Column 3} \\
    \midrule
    Data 1 & Data 2 & Data 3 \\
    \bottomrule
  \end{tabular}
\end{table}
```

### Cross-References

Follow these conventions:
- Chapters: `\ref{ch:<prefix>}` → "Chapter 1"
- Sections: `\ref{sec:<prefix>_<slug>}` → "Section 1.1"
- Figures: `\ref{fig:<prefix>_<slug>}` → "Figure 1"
- Tables: `\ref{tab:<prefix>_<slug>}` → "Table 1"

Use `Figure~\ref{fig:...}` (with tilde for non-breaking space) in running text.

### Code Listings

For inline code snippets, use:

```latex
\begin{lstlisting}[language=Python, caption={<Caption>}, label={lst:<label_prefix>_<slug>}]
# Your code here
def example():
    return "Hello"
\end{lstlisting}
```

For short inline code, use `\texttt{<code>}`.

### Section Transitions

End each section with a brief transition sentence leading to the next section. End the chapter with a conclusion paragraph summarizing the chapter and previewing the next.

```latex
\section{Conclusion}

In this chapter, we have <summarized contributions>. The next chapter will focus on <preview of next chapter topic>.
```

## Constraints

1. **Label naming**: All labels must use the format `<type>:<label_prefix>_<descriptive_slug>` where slugs are lowercase with underscores
2. **Box usage guidelines**:
   - `emeraldbox` → definitions, key results, system capabilities
   - `goldbox` → highlights, future work, planned features
   - `warnbox` → warnings, limitations, known issues
   - `infobox` → general context, informational notes
   - `phaseblock` → process steps, pipeline stages, workflows
3. **Float placement**: Always use `[H]` (requires `float` package) for figures and tables to prevent unexpected placement
4. **Figure placeholders**: Use `\fbox{\parbox{...}}` for placeholder figures — never leave empty `\includegraphics` calls
5. **No hardcoded page numbers** — always use `\ref{}` for cross-references
6. **Academic English tone** throughout — formal, third-person, present tense for descriptions
7. **Each section should be 200-400 words** minimum when generating actual content
