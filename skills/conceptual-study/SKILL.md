```latex
\label{sec:cs_uml}

\begin{emeraldbox}[title=Unified Modeling Language (UML)]
UML (Unified Modeling Language) is a standardized graphical modeling language used to visualize, specify, construct, and document the artifacts of a software system. It provides a common vocabulary for object-oriented analysis and design, supporting multiple diagram types for different perspectives of the system.
\end{emeraldbox}

The diagrams used in this project include:
\begin{itemize}
  \item \textbf{Use Case Diagrams} — to identify actors and their interactions
  \item \textbf{Sequence Diagrams} — to illustrate message flow for key scenarios
  \item \textbf{Class Diagram} — to define the static data structure
\end{itemize}
```

```latex
\begin{infobox}[title=Relational Schema]
\begin{itemize}
  \item \textbf{TableName}(\underline{id}, attribute1, attribute2, \textit{foreign\_key\_id})
  \item \textbf{AnotherTable}(\underline{id}, name, \textit{table\_name\_id})

\end{infobox}
```

## Constraints

1. **ALL diagrams must be generated by including external image files** using the `\includegraphics` command.
2. **Requirements tables must use IDs** (FR-01, NFR-01) for traceability.
3. **Minimum diagrams**: 1 global use case, 3 sequence diagrams, 1 class diagram, 1 architecture diagram.
4. **Describe each diagram** before presenting the figure, following the pattern: "The following diagram (Figure X) illustrates...".
5. **Provide a description table** for the Class Diagram elements.
6. **Database schema** must clearly distinguish primary keys (underlined) and foreign keys (italicized).
7. **Actor identification table** must precede use case diagrams to define all system actors.
